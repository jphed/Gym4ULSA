## Experiencia durante este Sprint

Mi experiencia en este Sprint fue buena. Como nos dividimos la carga de trabajo de manera aleatoria, siento que la distribución quedó más equilibrada. Además, el hecho de pedirle a la IA que la repartiera equitativamente ayudó bastante. Fue una experiencia agradable, especialmente considerando que antes, en ocasiones, una persona terminaba haciendo más trabajo que otras, lo cual me parecía injusto tanto para quien hacía de más como para quien aportaba menos.

## Aportaciones personales al equipo

Mis contribuciones fueron las siguientes:

- Código limpio y funcional.
- Integración de pantallas.
- Navegación sin errores.
- Eliminación de archivos innecesarios (como carpetas de macOS, por ejemplo `.DS_Store`).
- Consistencia general del proyecto (nomenclatura en inglés, vistas correctamente nombradas).

## Aprendizajes técnicos

Aunque no fue algo estrictamente técnico, aprendí la importancia de hacer las cosas bien desde el inicio. El proyecto de GitHub lo había creado en una carpeta incorrecta, lo cual generó complicaciones que pudieron evitarse si todo se hubiera configurado adecuadamente desde cero.

También reforcé buenas prácticas como definir el idioma en el que se nombrarán los archivos y comunicarlo al equipo. Yo siempre utilizo inglés, pero algunos compañeros lo hacían en español. Creo que para la próxima vez vale la pena enfatizar este punto para mantener la consistencia.

## Dificultades encontradas o resueltas

Relacionado con lo anterior, una de las principales dificultades fue la falta de buenas prácticas desde el comienzo. Es mucho más sencillo establecer parámetros claros y consistencia desde el inicio que tener que hacer correcciones a mitad del desarrollo.

## Buenas prácticas de Git aplicadas (ejemplos concretos)

Principalmente, el uso de ramas para evitar afectar la rama "main" y la creación de pull requests para integrar cambios. Aunque a veces se nos olvida a mí incluido crear una rama para cada tarea, en general hacemos un buen uso de esta práctica.

## Compromisos para la entrega final del proyecto

Me comprometo a desarrollar la API REST con Apollo Server y a integrar la comunicación del proyecto a través de esta API, en lugar de hacerlo mediante gists.
